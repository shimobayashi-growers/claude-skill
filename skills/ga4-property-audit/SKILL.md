# GA4プロパティ設定チェック

## ワークフロー

```
1. プロパティ設定・カスタム定義・Ads連携を取得
   ↓
2. 推奨構成との乖離をチェック
   ↓
3. 問題点・推奨改善をレポート出力
```

---

## Step 1: データ取得

以下の3つのMCPツールを**並列で**呼び出す。

- **property_id:** 環境変数 `GA4_PROPERTY_ID` から取得。未設定の場合はユーザーに確認する

### 1-1. プロパティ詳細

`mcp__analytics-mcp__get_property_details` を使用。

```json
{
  "property_id": "<GA4_PROPERTY_ID>"
}
```

### 1-2. カスタムディメンション・メトリクス

`mcp__analytics-mcp__get_custom_dimensions_and_metrics` を使用。

```json
{
  "property_id": "<GA4_PROPERTY_ID>"
}
```

### 1-3. Google Ads連携

`mcp__analytics-mcp__list_google_ads_links` を使用。

```json
{
  "property_id": "<GA4_PROPERTY_ID>"
}
```

---

## Step 2: チェック項目

取得したデータを以下の観点でチェックする。

### 2-1. プロパティ基本設定

| チェック項目 | 推奨設定 | リスクレベル |
|---|---|---|
| データ保持期間 | 14ヶ月 | 🔴 2ヶ月のままだとデータ損失 |
| データストリーム設定 | Web + 拡張測定ON | 🟡 測定漏れの可能性 |
| タイムゾーン | Asia/Tokyo | 🟡 レポート日付ズレ |
| 通貨 | JPY | 🟡 収益データ不正確 |
| Google シグナル | 有効推奨 | 🟡 クロスデバイス分析制限 |

### 2-2. カスタム定義の品質

| チェック項目 | 判定基準 | リスクレベル |
|---|---|---|
| 命名規則の統一 | snake_case推奨、ばらつきがないか | 🟡 管理困難 |
| PII漏洩リスク | email, phone, name等が含まれていないか | 🔴 ポリシー違反 |
| 未使用の定義 | スコープに対してデータが空でないか | 🟡 枠の無駄遣い |
| スコープの適切性 | EVENT/USER/ITEMの使い分け | 🟡 分析制限 |
| 上限への近さ | イベント50個、ユーザー25個の枠 | 🟡 拡張性制限 |

### 2-3. Google Ads連携

| チェック項目 | 推奨設定 | リスクレベル |
|---|---|---|
| 連携有無 | 広告運用中なら必須 | 🟡 コンバージョン計測不可 |
| 自動タグ設定 | ON推奨 | 🟡 アトリビューション不正確 |

## 出力フォーマット規約

- 数値はカンマ区切り（例: `1,234`）
- パーセンテージは小数点1桁（例: `89.8%`）
- 根拠データは全て GA4 API レスポンスから直接引用。推測値には「推定」と明記
- テーブルの数値は右寄せ（`---:`）

## 注意事項

- プロパティ設定の一部はGA4管理画面でのみ確認可能。APIで取得できない項目は「API未対応」と明記する
- PII判定は命名ベースの推定。実データの中身までは確認できないため「要確認」と記載する
- ハルシネーション防止: 数値は必ずAPIレスポンスから直接引用し、計算値は計算過程を示す
