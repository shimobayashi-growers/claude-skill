# 未返信Slackメンション確認

## ワークフロー概要

```
1. 自分のSlackプロフィール取得（ユーザーID特定）
   ↓
2. 対象日の決定（デフォルト: 昨日）
   ↓
3. 対象日のメンションを検索
   ↓
4. 各メンションのスレッドを確認し、返信済みか判定
   ↓
5. 未返信メンションを整形して表示
```

## Step 1: ユーザーID取得

`slack_read_user_profile` を **user_id なし** で呼び出し、自分のプロフィールを取得する。レスポンスからユーザーID（例: `U01ABCDEF`）と表示名を取得する。

## Step 2: 対象日の決定

- デフォルト: **昨日**の日付を使用する
- ユーザーが日付を指定した場合はその日付を使用する（例: 「3月27日のメンション確認して」）
- 対象日を `TARGET_DATE`（YYYY-MM-DD）、翌日を `NEXT_DATE`（YYYY-MM-DD）として保持する

## Step 3: メンション検索

`slack_search_public_and_private` を使用して、対象日に届いたメンションを検索する。

- **検索クエリ**: `<@USER_ID> after:TARGET_DATE before:NEXT_DATE`
  - 例: `<@U01ABCDEF> after:2026-04-01 before:2026-04-02`
- 検索結果からメッセージ一覧を取得する

**注意**: `slack_search_public_and_private` でエラーが出た場合は `slack_search_public` にフォールバックし、公開チャンネルのみの結果であることをユーザーに伝える。

## Step 4: 未返信判定

各メッセージについて以下を確認する:

1. **自己メンション除外**: メッセージの送信者が自分自身の場合はスキップする
2. **スレッド確認**: `slack_read_thread` でスレッドを取得する
3. **返信判定**: スレッド内の返信者に自分のユーザーIDが含まれるか確認する
   - 含まれていない → **未返信**として記録
   - 含まれている → スキップ
4. **スレッドなし**: メッセージにスレッドがない場合（返信がゼロ）は**未返信**とみなす

## Step 5: 結果出力

### 未返信メンションがある場合

```
【未返信メンション】YYYY年M月D日（曜日）分

1. #channel-name | sender名 | HH:MM
   「メッセージプレビュー（50文字以内）...」
   URL: https://slack.com/archives/CHANNEL_ID/pTIMESTAMP

2. #channel-name | sender名 | HH:MM
   「メッセージプレビュー...」
   URL: https://slack.com/archives/CHANNEL_ID/pTIMESTAMP

合計 N件
```

### 未返信メンションがない場合

```
YYYY年M月D日（曜日）の未返信メンションはありません 🎉
```

## 注意事項

- 検索結果が大量（50件超）の場合は、チャンネル別にグループ化して表示する
- メッセージプレビューは50文字以内に切り詰め、`...` を付与する
- URLはSlackのメッセージリンク形式（`https://slack.com/archives/...`）で出力する
- 曜日は日本語表記（月・火・水・木・金・土・日）を使用する
